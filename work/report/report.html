1. Reddit

Reddit is a website I use everyday for a wide variety of reasons. I would describe it as a modern looking forum where you subscribe to a topic/genre eg. /r/soccer if i'm interested in soccer. I have always viewed the site to be a little slow but not at all problematic so measuring it to see if it actually is slow compared to real metrics could be interesting.

The measured performance turned out to be worse than I expected and some parts of the tests showed unreasonable results although that could be faulted by my understanding of the metric. Lighthouse and Pagespeed showed a similar general performance score of 20 and 26 but quite different results when it gets specific. Lighthouse showed a Time to interact of 28.8 seconds while Pagespeed showed a time of 11.1 seconds. None of the results feels representative of how it actually feels using the website. I would say it feels like 2-3 seconds when using it and i thought that was really slow so there might be some kind of issue when using these tools for this particular metric. Overall Pagespeed gave way faster results compared to Lighthouse yet gave it a similar general score.

Using the network tab i could see that it's a quite heavy site compared to the other 2 sites we analysed with up to 5 times the amount of resources received measured in MB. This is probably the result of the many images that are being loaded on reddit and this is probably also the reason the site takes 24.13 seconds to finish loading the page.

To speed up the page it's recommended to properly size the images which would lead to less data usage. It is also recommended to minimize main thread work but it's unclear on just how that would be done.

Web Performance will always be an important part of web development and it’s something to take seriously. Web pages today are expected to be fast these days and having a slow and unresponsive experience can have a serious impact on the user base.

2. Hltv

I chose the website www.hltv.org. The reason being since it's a website I often use, I would say daily. I’ve noticed it can be a bit slow sometimes and It could be interesting to evaluate the performance of a website I often visit.

We could say that www.hltv.org could definitely improve their performance. However, it is a bit confusing since the PageSpeed Insights score is 98 whereas the Lighthouse speed is 35. I made sure that the tests were on desktop and I tested it in regular-mode and incognito-mode, still the same results. The speed index on Lighthouse was 6.8s whereas the speed index on PageSpeed Insights was 1.5s. It is a bit confusing since I don’t know which one is “correct”. Time to interact was also off. On PageSpeed Insights it was 1.7s and on Lighthouse it was 9.0s.

Regarding the network tabs findings, the requests was 102 which compared to the other two websites, was the lowest. The resources were only 4.6 MB as well which, compared to the other websites, were the lowest.

In order to load the page faster, they could properly size their images in order to save cellular data and this will improve the load time. They could also eliminate render-blocking resources.

Web testing is important in order to check your webpage for potential bugs before its made live and is accessible to the public. When testing performance, you test the webpages response times at different connection speeds, stress test your website to determine its break point, test if a crash occurs due to peak load and also test the optimization for the browser and server side cache to reduce load times and compressions.
